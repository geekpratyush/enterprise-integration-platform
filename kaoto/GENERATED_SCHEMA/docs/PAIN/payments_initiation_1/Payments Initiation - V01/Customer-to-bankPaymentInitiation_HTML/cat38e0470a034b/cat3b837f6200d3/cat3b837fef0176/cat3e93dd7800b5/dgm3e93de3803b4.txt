Scenario : request for cancellation of group.

The initiating party submits a payment initiation to the first agent, containing several single payment transactions.
The initiating party then requests the first agent to cancel the entire payment initiation.
In theory, a payment order is revocable till it is processed by the first agent, ie it becomes irrevocable once it is sent in the interbank chain by the first agent. More precisely, the cancellation is possible up to the warehousing in the activity diagram. Nevertheless, it depends on the system capacity and on the country laws (legal requirements).  It's also important to note that a request for cancelling payment orders must follow the same rules for authorization as the payment orders themselves.

In this scenario, the first agent can accept the request to cancel the group.

1. Information needed by the initiating party to request the cancellation of the entire payment initiation sent 
New reference for cancellation request, Group ID (of original payment initiation), requested execution date, possibly payment type identification details and name of original message type sent, debtor's account details and debtor identification.

2. Information needed by the first agent to link the request to the original payment iniation received :
Group ID (of original payment initiation), requested execution date, possibly payment type identification details and name of original message type sent, debtor's account details and debtor identification.

3. Information needed by the initiating party to reconcile the request for cancellation status to the request for cancellation :
Reference of the cancellation request, group ID (of original payment initiation), requested execution date, possibly payment type identification details and name of original message type sent, debtor's account details and debtor identification.

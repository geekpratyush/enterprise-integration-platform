Activity Forward execution status

Parent State/ActivityNone  


Documentation
The forwarding agent forwards to the initiating party a notification received from the first agent that the payment order file has been executed. Depending on the payment method requested and depending on the routing information contained in the payment order :
· the creditor(s) has/have been credited in the books of the first agent (for 'on-us' payments)
· the first agent has printed and sent cheques
· the first agent has sent off the payment orders which need to be further routed in the payment chain (for 'not on-us' payments)

Note :
· if the creditor has an account in the books of the first agent (ie the agent of the debtor), this execution status is a 'final' execution status (ie the creditor has been credited by his agent).
In the other cases ('receive cheques'/and when the creditor has an account at a different agent than the first agent), the execution status is not yet 'final' - ie the execution status does not include the actual confirmation of credit of the creditor by his agent - or cashing of cheque by the beneficiary.
· the activity diagram shows that the first agent could either send the execution status directly to the initiating party - or through the forwarding agent (if any), as is actually the case for all 'status' information. This would depend on the 'solution' and 'agreements' chosen and agreed between first/forwarding/initiating parties.

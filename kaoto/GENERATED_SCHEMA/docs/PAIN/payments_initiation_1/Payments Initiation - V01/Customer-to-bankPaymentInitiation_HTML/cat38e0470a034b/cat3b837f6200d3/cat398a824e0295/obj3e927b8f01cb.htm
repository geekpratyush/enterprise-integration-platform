Object FirstAgent : FinancialInstApplication {Analysis}

Documentation
In this scenario, the first agent receives a group with multiple single transactions.
The first agent rejects the complete group as the percentage of individual faulty transactions has exceeded a pre-agreed treshold between initiating party and first agent.

The first agent rejects the complete group and sends a status message to inform the initiating party, referring to the original group and indicating the reason for reject of the group.

Information needed by the initiating party to reconcile the reject with payment order sent :
Group ID, requested execution date, possibly payment type identification details, name of message type sent, debtor's account details and debtor identification.

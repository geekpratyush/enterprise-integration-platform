Documentation
Internal security procedures are applied by the forwarding/first agent in order to authenticate the payment orders (meaning identifying the sender of the payment orders and checking the integrity of the file : ie checking that the payment information has not been changed during transmission). In case authentication fails upon receipt of the file, the receiving agent may investigate whether he has used correct authentication information (eg correct bilateral keys) and re-try the authentication process once more before rejecting the file.

Scenario : request for cancellation of individual transaction in a group.

The initiating party submits a payment initation to the first agent, containing several single payment transactions.
The initiating party then requests the first agent to cancel 2 individual  payment transactions out of the group.
 
In this scenario, the first agent can accept the request to cancel the individual transactions.

1. Information needed by the initiating party to request the cancellation of 2 individual payment transactions in the group
New reference for cancellation request, Group ID (of original payment initiation), requested execution date, possibly payment type identification details and name of original message type sent, debtor's account details and debtor identification. + individual payment instruction ID/payment transaction ID/instructed amount + optionally creditor's account details and creditor ID.

2. Information needed by the first agent to link the request to the original payment iniation received :
Group ID (of original payment initiation), requested execution date, possibly payment type identification details and name of original message type sent, debtor's account details and debtor identification. + individual payment instruction ID/payment transaction ID/instructed amount + optionally creditor's account details and creditor ID.

3. Information needed by the initiating party to reconcile the request for cancellation status to the request for cancellation :
Reference of the cancellation request, group ID (of original payment initiation), requested execution date, possibly payment type identification details and name of original message type sent, debtor's account details and debtor identification + individual payment instruction ID/payment transaction ID/instructed amount + optionally creditor's account details and creditor ID.

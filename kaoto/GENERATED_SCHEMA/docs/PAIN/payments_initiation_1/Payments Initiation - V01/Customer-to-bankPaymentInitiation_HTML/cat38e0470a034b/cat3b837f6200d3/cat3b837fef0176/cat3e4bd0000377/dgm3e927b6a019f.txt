Scenario : reject of individual transaction in the group by first agent.

In this scenario, the first agent receives a group with multiple single transactions. 
The first agent rejects eg 1 individual transaction that contains an error, but accepts all other individual transactions. 

The first agent rejects the specific single faulty transaction and sends a status message to inform the initiating party, referring to the original group and the individual faulty transaction and indicating the reason for reject of the specific individual transaction.

Information needed by the initiating party to reconcile the reject of the individual transaction with group payment initiation sent :
Group ID, requested execution date, possibly payment type identification details and name of message type sent, debtor's account details and debtor identification.+ individual payment instruction ID/payment transaction ID/instructed amount + optionally creditor's account details and creditor ID.
